# i18n approach

Molotov uses a lightweight dictionary under `apps/web/lib/i18n/` instead of `next-intl` for this first ES/EN pass.

The current site does not need locale-prefixed routes, server-side negotiation, plural rules, or message extraction yet. A small React context keeps the App Router surface simple while still giving us instant client-side language switching, persisted locale state, and typed translation keys.

- ES is the default locale.
- EN lives beside it as a complete peer dictionary.
- The selected locale is stored in `localStorage` as `molotov:locale`.
- On mount and on change, the provider updates `<html lang>`, document metadata, and the manifest link (`/manifest-es.json` or `/manifest-en.json`).
- When PT lands, add `pt.ts`, extend the `Locale` union and dictionary map, then expose it in the nav switcher.
